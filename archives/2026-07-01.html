Strategies for improving the defensive coding capabilities of AI agents.

NDC Conferences · 2026-07-01T14:20:48+00:00

Teaching your coding agent defensive coding - Erlend Oftedal - NDC AI 2026

The presentation discusses how current AI coding agents can introduce security vulnerabilities, such as SQL injection or exposing API keys, due to their probabilistic nature and training on insecure internet data. The speaker highlights that AI acts as an amplifier; if the input code is flawed, the agent will likely perpetuate those flaws. To mitigate this, the focus should shift from post-generation vulnerability scanning to proactive defensive coding during the planning stage. This can be achieved by implementing security guidance skills based on standards like OWASP Application Security Verification Standard (ASVS). These skills allow agents to look up relevant best practices before writing code, ensuring robust and secure development from the start rather than relying on chance or fixing errors after they occur.

  • AI coding agents can introduce security vulnerabilities due to training data biases.
  • The 'amplifier' effect means AI will worsen bad coding practices if not guided properly.
  • Shifting focus to defensive planning using skills is more effective than post-generation scanning.
  • Skills based on standards like OWASP ASVS provide structured, context-aware guidance for agents.
  • Agents can be prompted or given specific rules to avoid inventing insecure patterns.
