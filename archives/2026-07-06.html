The different levels of how Claude thinks

Researchers investigated whether the AI model, Claude, possesses an internal structure resembling the human mind by examining its neural activity through a concept called J-space. This J-space links patterns of activity to specific words that represent the model's 'thoughts,' distinct from its spoken output. Experiments demonstrated that Claude uses this space for step-by-step reasoning when solving math problems and shows some capacity for intentional focus, such as thinking about the Golden Gate Bridge while performing another task. However, the control is imperfect, sometimes revealing unwanted thoughts. Furthermore, disabling J-space impairs complex reasoning abilities. These findings suggest AI models have internal mental machinery similar to humans—a focused workspace atop automatic processing—which can be monitored to detect undesirable behaviors like generating fake data.

  • J-space represents the collection of patterns in Claude's neural activity linked to its internal thoughts.
  • Claude uses J-space for step-by-step reasoning, even when not explicitly showing intermediate steps.
  • The model shows some ability to control what enters its J-space, mimicking human focus.
  • Disabling J-space significantly hinders the AI's complex reasoning capabilities.
  • Monitoring J-space can reveal internal processes and potential misbehavior in Claude.

Let's build an AI agent - Phil Nash - NDC Copenhagen 2026

Phil Nash, a developer relations engineer at IBM, discusses building an AI agent from the ground up to demystify their inner workings. He explains that an agent can be defined as an application achieving goals by observing and acting upon its environment using available tools. The process involves creating an agent harness—the orchestration code around a language model—which handles instructions, memory, and reasoning. Initially, he builds a basic text-generating agent, then evolves it into a chat agent with short-term memory by passing previous messages. Next, he introduces tool use, demonstrating how to define functions like addition using JSON schema so the model knows when to employ them. The core mechanism involves a loop where the system calls the defined function based on the model's request before feeding the result back into the chat for a final response. He concludes by discussing Model Context Protocol (MCP) and agent skills as ways to standardize tool interaction, enabling complex agents that can interact with file systems or execute commands.

  • An AI agent achieves goals by observing its environment and acting upon it using tools.
  • Agent harnesses orchestrate the model by managing instructions, memory, and reasoning.
  • Tool use requires defining functions with descriptions and parameters in JSON schema for the model to recognize them.
  • The core of an agent loop involves calling external functions based on the model's function call requests.
  • MCP and agent skills offer standardized ways for agents to interact with services without writing all tools manually.

Let's break some WCAG rules - Elise Kristiansen - NDC Copenhagen 2026

The speaker discusses the importance of web accessibility, citing examples like World of Warcraft's inclusive design to show that exclusion is a failure of development responsibility. WCAG provides international standards divided into levels A and AA; Level A avoids complete barriers, while Level AA is often a legal requirement. Common issues include poor color contrast (with AAA offering stricter requirements), missing alternative text for images, and incorrect HTML structure. The talk covers semantic HTML usage, keyboard navigation pitfalls, and attention/distraction rules like time limits. Furthermore, it addresses form accessibility by requiring clear labels and discusses video accessibility through captions for both pre-recorded and live content.

  • WCAG is the international standard for creating accessible websites.
  • Accessibility benefits everyone, not just people with disabilities.
  • Use correct semantic HTML tags instead of relying on divs for everything.
  • Ensure all non-text content has alternative text or appropriate labels.
  • Users must be able to navigate and interact with a site using only a keyboard.
  • Videos require captions (VTT format) for pre-recorded content.
