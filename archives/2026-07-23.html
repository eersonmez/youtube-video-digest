The misconception that Big O notation is the most important factor in algorithm performance.

NDC Conferences · 2026-07-23T10:00:25+00:00

The Biggest Misconception of Computer Science - Alex Dathskovsky - NDC Toronto 2026

The speaker argues that relying solely on Big O notation, which treats every operation as having a cost of one, overlooks crucial real-world factors like hardware architecture. While theoretical analysis shows complexities such as quadratic time or n log n, actual execution speed depends heavily on how the algorithm interacts with the CPU and memory hierarchy. Concepts like cache friendliness, instruction set efficiency (e.g., using shifts instead of division), SIMD vectorization, and branch prediction significantly impact runtime. The talk demonstrates that an algorithm with a theoretically worse Big O complexity can outperform a seemingly superior one if it is better optimized for the underlying hardware.

  • Big O notation assumes isolated algorithms without considering hardware constraints.
  • Hardware factors like CPU architecture, cache size, and memory access dictate real-world performance.
  • Techniques such as SIMD vectorization and branchless computing optimize code for modern processors.
  • Cache friendliness is vital; linear data structures like vectors can outperform hash tables due to better cache line utilization.
  • Amortized analysis provides a more realistic view of operations that might occasionally incur high costs.
