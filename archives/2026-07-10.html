Put an Agent Inside Your App in 10 Minutes or Less with the GitHub Copilot SDK - Daniel Ward

The talk introduces the concept of an AI agent, clarifying that at its core, it is a Large Language Model (LLM) augmented with tools and looping capabilities. The presenter details how LLMs can be enhanced by giving them access to external functions, such as browsing websites or running code, rather than relying solely on their training data. The GitHub Copilot SDK allows developers to embed this agent functionality into their applications quickly. Demonstrations show initial setup in under ten minutes, followed by advanced use cases like automatically suggesting fixes for production errors or creating daily briefs by aggregating information from Jira and Teams. While prompt chaining is easy to prototype, the SDK offers a more reliable path for repeatable tasks because it allows deterministic code to handle non-creative steps, ensuring higher success rates than relying solely on AI.

  • An agent is fundamentally an LLM enhanced with tools and looping.
  • The GitHub Copilot SDK enables embedding agent capabilities into applications quickly.
  • Agents can be used for complex tasks like suggesting code fixes from production logs.
  • Using the SDK provides more reliability than pure prompt chaining due to deterministic code integration.
  • Production deployments require sandboxing, such as using Docker, to limit AI access.
