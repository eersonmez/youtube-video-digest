The Patron Saint of Murderers - Ada Palmer

St. Julian the Hospitaler was a popular Renaissance saint who is considered the patron saint of murderers. His legend involves a curse from a witch that prophesied he would slay his parents upon growing up. After fleeing and feeling homesick, he was tricked into committing the act. He spent the remainder of his life seeking atonement through pilgrimage and dedicating himself to guiding others in their spiritual journeys. This contrasts sharply with modern cultural attitudes toward homicide, which often dictate that murderers should be shunned or imprisoned without possibility of redemption. The Renaissance perspective emphasized that after committing a sin like murder, what mattered was the subsequent remorse and the need for spiritual guidance.

Dangerous Reactivity: Why AI Output Is the New XSS - Ramona Schwering - NDC Copenhagen 2026

Ramona Schwering presents a talk on XSS, highlighting that the integration of LLMs has created a 'new XSS' threat. This occurs when an attacker uses prompt injection to manipulate an LLM into generating malicious code or payloads, which is then passed downstream and trusted by the application backend. The core danger lies in improper output validation and sanitization of this AI-generated content. Attacks can manifest as classical XSS (stealing user sessions), SQL injection (e.g., dropping tables), Remote Code Execution (RCE) via code execution features, or even through Markdown parsers that allow HTML injection. To mitigate these risks, the speaker advises adopting a zero-trust mindset for AI output, using context-aware encoding, employing layered defenses like input validation and Content Security Policy, and strictly enforcing the principle of least privilege on LLM interactions.

  • LLM output can act as an intermediary to build malicious scripts, combining prompt injection with classic injection attacks.
  • Improper handling of LLM output leads to new XSS vectors across various contexts like DOM, SQL, and shell commands.
  • Mitigation strategies include using structured JSON outputs instead of raw text and implementing sandboxing for code execution.
  • A zero-trust mindset must be applied to all AI inputs and outputs, treating them as untrusted external sources.
  • Layered security approaches involving input validation, CSP, and monitoring are necessary for defense.
