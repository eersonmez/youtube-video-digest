Building a real-time collaborative editor - Mihaela Gheorghe-Roman - NDC Copenhagen 2026

This presentation details the development of a collaborative text editor designed specifically for disconnected, airgapped military and defense systems. The core challenge involved enabling multiple users across different locations to work on documents in real time without internet connectivity, requiring data merging when servers reconnect after long periods offline. The solution moves beyond basic rich text editing by implementing complex features like granular security clearance access down to specific paragraphs or links. To ensure consistency during concurrent edits—such as insertions and deletions—the system relies on mathematical concepts of commutativity and idempotency, using rules like 'last man wins' for conflicts. Advanced operations include renaming, splitting paragraphs, and handling object manipulations within a tree-based structure, all while maintaining security protocols.

  • The editor is designed to function in airgapped and disconnected environments.
  • Conflict resolution relies on mathematical concepts of commutativity and idempotency.
  • Security clearance can be applied granularly down to specific document sections.
  • Operations are managed through custom JSON objects called actions or operations.
  • The system supports advanced features like undo/redo via a stack manager.

The speaker discusses evolving software engineering practices, moving away from outdated architectural patterns toward more modern, human-centric approaches.

Stop writing software like it's 1999! - Hannes Lowette - NDC Copenhagen 2026

Hannes Lowette critiques traditional software development methodologies, noting that many current practices resemble those from the late 1990s. He argues against rigid layered architectures because they force developers to navigate numerous empty blocks across layers for a single feature, increasing friction and context switching. Instead, he advocates grouping code by feature, which allows for some duplication but significantly improves human readability and developer productivity in agile environments. Furthermore, the talk addresses microservices, suggesting that small teams owning slices of a monolith is often preferable to managing dozens of services unless scaling demands it. The speaker concludes by emphasizing the necessity of robust testing and modern data patterns like Command Query Responsibility Segregation (CQS) over strictly normalized databases to reduce friction and enable rapid iteration.

  • Avoid rigid layered architectures that increase development friction.
  • Group code by feature rather than technical layer for better human reasoning.
  • Microservices are beneficial for large organizations but can introduce unnecessary complexity.
  • Automated testing is crucial for building developer confidence and enabling faster deployment cycles.
  • Modern data patterns like CQS reduce reliance on complex database locking mechanisms.
