How I use LLMs

Karpathy gives a broad practical walkthrough of the LLM product ecosystem rather than another under-the-hood model lecture. He starts with ChatGPT and its peers, explaining that models combine pretraining knowledge with post-training assistant behavior and that users should pay attention to model choice, plan tier, and whether a task needs a fast non-thinking model or a slower thinking model. He then emphasizes tool use: search is best for recent, niche, or changing information; uploaded documents give the model concrete context; and outputs should be treated as first drafts that still need checking. He demonstrates calculator/Python-style tool use for math, Claude-style artifacts for generating small custom apps, and agentic coding tools such as Cursor for iterative software work, while noting that review remains necessary. Later sections cover voice/audio interactions, NotebookLM-style podcast generation, image and camera understanding, custom GPTs, saved instructions, memory, few-shot prompts, and the differences between web and mobile app features. His overall advice is to experiment across tools, choose stronger models for professional work when the cost is justified, provide concrete examples, and avoid blindly trusting generated tables, code, or claims.

  • Model selection matters: cheap/fast models are often enough, but professional coding and hard reasoning can justify paying for stronger models.
  • Use search when information is recent, obscure, or likely to change; use uploaded documents when the answer must reference specific context.
  • Treat LLM output as a first draft and check it, especially for tables, numbers, code, and factual claims.
  • Tool use extends models beyond next-token text: Python handles arithmetic/code execution, artifacts can create custom local apps, and coding agents can edit projects iteratively.
  • Multimodal features are now part of the workflow: voice, audio/podcast generation, image input, and live camera understanding all have different strengths and product limits.
  • Reusable prompts, custom GPTs, instructions, memory, and few-shot examples save repeated setup and improve task consistency.
