Performing just twenty bodyweight squats each day can yield significant physiological changes, affecting everything from muscle mass to brain health. Guidelines for the responsible abandonment of open source projects.

NDC Conferences · 2026-07-07T09:57:47+00:00

Responsibly Abandoning Open Source Projects - Jason Turner - NDC Copenhagen 2026

Jason Turner discusses best practices for maintaining and eventually abandoning open source projects, emphasizing quality over quantity in development. He advocates for a project to be proven, maintainable, resumable, and provable. Key technical recommendations include using automatic code formatters, ensuring build/test processes are simple (ideally under five lines), and implementing rigorous testing strategies like line, branch, and path coverage. Furthermore, he stresses the importance of runtime analysis tools such as sanitizers and mutation testing to validate test suites. When deciding to stop maintenance, it is advised not to simply archive a project on GitHub unless there is a proven successor fork, suggesting instead that ownership be held until the decision is final.

  • Projects must be proven, maintainable, resumable, and provable.
  • Implement automatic code formatting to reduce pull request friction.
  • Use comprehensive testing including line, branch, and path coverage.
  • Employ runtime analysis tools like sanitizers and mutation testing.
  • Only archive a project if a clear successor fork exists.
