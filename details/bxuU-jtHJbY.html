Integrating AI capabilities into mobile applications using .NET MAUI.

NDC Conferences · 2026-07-07T09:29:27+00:00 · Building an AI-Powered Personal Companion Mobile App - Gerald Versluis - NDC Copenhagen 2026

What This Video Is About

This video details how developers can integrate Artificial Intelligence capabilities into custom mobile applications using the .NET MAUI framework. The central theme is moving beyond simple to-do lists to create intelligent, context-aware personal companions that can handle complex user needs through multimodal interaction. By leveraging AI services, developers can build features such as personalization based on user habits, understanding inputs from voice or images, and reacting intelligently to the user's environment.

The presentation emphasizes that while AI offers immense power for creating advanced applications—like an app advising on food allergies based on ingredients or analyzing photos of menus—it must be implemented responsibly. A key guideline stressed is avoiding "creepy" user experiences by being mindful of how much personal data and context the application collects and utilizes.

Detailed Walkthrough

The session begins by establishing that AI can be incorporated into any .NET project, but mobile applications built with .NET MAUI are particularly suited for these advanced features [0:00]. The speaker introduces Microsoft Extensions AI as the mechanism to connect applications with various AI models.

A significant discussion revolves around responsible AI; a primary ground rule is "don't be creepy" when dealing with user data and personalization [12:34]. Personalization can become powerful, allowing an AI to build a profile of a user based on conversations (e.g., estimating age or location), which can then inform advice, such as tailoring workout schedules for a specific demographic [15:00].

The video explores several advanced interaction types enabled by mobile devices and AI:

  • Context Awareness: Mobile devices possess various sensors that allow applications to know the user's location (even without GPS via SIM triangulation), calendar events, and environmental factors like ambient light [10:00]. This context can drive proactive assistance, such as prompting a user before an important meeting based on their travel plans.
  • Multimodal Interaction: AI excels at processing different types of input beyond text. Users can interact by sending pictures for description or translation, or by speaking to the application (voice interaction) [10:00]. While voice assistants like Siri have historically required very specific phrasing, the goal is achieving more natural conversation flows [10:00].
  • Memory and Context: AI models can build a memory of past interactions, allowing them to recognize themes or preferences over time. However, users retain control and can instruct the model to forget specific details [15:00].

The technical implementation relies on abstraction layers provided by .NET MAUI. For UI elements (like an activity indicator), MAUI abstracts platform-specific implementations (e.g., iOS vs. Android progress bars) into a unified concept, ensuring consistent behavior across platforms while respecting native look and feel [15:00].

For AI integration, the speaker highlights two main libraries: Microsoft Extensions AI and Vector Data, which handle common AI abstractions [20:00]. A major advantage of these frameworks is the ability to swap concrete implementations. Developers can use models from Microsoft Foundry (leveraging frontier models) or utilize local models available on platforms like iOS via Apple Intelligence by simply registering a different implementation without changing core code [15:00].

The demonstration showcases an application designed to help users navigate food allergies, specifically sesame allergies, in Denmark [23:00]. The app supports three input methods: text, voice, and image. When using the image feature, the user can take a photo of a menu or label; the process involves capturing the image via the built-in .NET MAUI Media Picker API, loading it as a file, sending the image bytes to the AI for analysis (using an `analyze async` method), and receiving structured advice regarding allergens [45:00].

For voice input, the app utilizes community plugins like plug-in.mmau.audio to record audio. This recorded file is then processed by a speech-to-text service (like Whisper) to generate transcribed text, which is subsequently fed into the main AI chat client for allergen analysis [40:00].

Finally, the speaker discusses running these applications locally using Onyx runtime, allowing sensitive data processing on the device without sending it over the internet, which is crucial for scenarios like hospital clinical support systems [50:00].

Key Takeaways

  • AI capabilities can be integrated into mobile apps developed with .NET MAUI.
  • Microsoft Extensions AI provides a unified interface to connect applications with various AI models and services.
  • Applications can leverage context awareness by integrating device APIs for location, calendar data, and environmental sensors.
  • Multimodal interaction allows users to interact via text, voice (using audio plugins), or images (analyzing photos of menus/labels).
  • Responsible AI mandates that developers actively prevent their applications from becoming intrusive or "creepy" with user data collection.
  • The framework supports swapping underlying AI implementations—allowing use of cloud models like those from Microsoft Foundry or local models via Onyx.
  • For image analysis, the process involves capturing media using .NET MAUI APIs, loading it as a file, and sending the byte stream to the AI for processing.
  • Voice input requires recording audio with community plugins, transcribing that audio into text using a service like Whisper, and then passing the resulting text to the LLM.

Practical Notes

  • Framework: .NET MAUI is used as the primary application framework.
  • AI Integration: Microsoft Extensions AI provides the core abstraction layer for interacting with AI services.
  • Model Providers: Developers can utilize models from Microsoft Foundry, or local models via Onyx runtime (for on-device processing).
  • Implementation Steps (Image Analysis): Use the built-in .NET MAUI Media Picker API to capture/select a photo -> Load the photo asynchronously into a file -> Send the image byte stream to the AI endpoint for analysis (`analyze async`).
  • Implementation Steps (Voice Analysis): Record audio using plugins like plug-in.mmau.audio -> Stop recording and get the file path -> Process the audio file asynchronously via a speech-to-text service (e.g., Whisper) -> Pass the resulting transcribed text to the AI chat client for analysis.
  • System Prompt: A system message can be defined in the model configuration (or dynamically created) to set the AI's persona and function, such as defining it as a "food allergy assistant."
  • Risk/Caveat: Sending large images or long audio files can consume a significant number of tokens; developers should optimize input size where possible.
