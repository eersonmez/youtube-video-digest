The presentation details how to build and understand AI agents by constructing one from scratch.

What This Video Is About

This video provides a detailed, hands-on demonstration of how to construct and understand an Artificial Intelligence (AI) agent from the ground up. The central theme is demystifying the inner workings of modern AI agents by building one piece by piece, moving from a simple text generator to a sophisticated system capable of memory, tool use, and self-improvement.

The speaker defines an AI agent as an application that achieves goals by observing its environment and acting upon it using available tools. The core mechanism driving this is the "agent harness"—the orchestration code surrounding a Large Language Model (LLM)—which manages instructions, memory, and reasoning through iterative loops. By building this from scratch, the audience learns the fundamental components required to move beyond simple text generation into autonomous goal-seeking behavior.

The presentation covers several advanced concepts necessary for robust agents, including defining tools via JSON schema, implementing a core execution loop, and leveraging standardized protocols like Model Context Protocol (MCP) and Agent Skills to enable complex interactions with external services without manually coding every capability.

Detailed Walkthrough

The demonstration begins by establishing the theoretical foundation of an AI agent. An agent is defined as an application that achieves goals by observing its environment and acting upon it using tools [0:15]. The speaker notes that while some definitions emphasize perception, a more fundamental view is that an LLM agent runs tools in a loop to achieve a goal [3:20]. Tools are crucial because they bridge the gap between the LLM's internal text generation capabilities and its ability to impact the external world [4:15].

The construction starts with a basic JavaScript application acting as the agent harness. Initially, this is an "echo agent" that simply repeats input [7:00]. This is then upgraded to a "basic agent" by hooking it up to an LLM (specifically Gemini 3 Flash preview) and providing system instructions, such as being a helpful assistant [9:15]. However, at this stage, the agent suffers from immediate amnesia because no memory mechanism is in place [10:00].

To introduce short-term memory, the agent is evolved into a "chat agent." The secret to this memory is passing all previous messages—both user prompts and model responses—back into the LLM with every new request. This process allows the agent to maintain context about past interactions, which is vital for tracking progress toward a goal [10:35].

The next major step is introducing tool use, which gives the agent agency over its environment. The speaker demonstrates this by creating an "add" function. For the LLM to know when and how to use this function, it must be declared using JSON schema, specifying the function's name, a description of its purpose, and its parameters (e.g., two numbers, 'a' and 'b') [15:00].

The core agent logic is then built by creating an "agent agent." This involves implementing a large `while` loop that continues as long as the model requests function calls [18:30]. Inside this loop, the system iterates over all requested function calls, executes the corresponding external JavaScript functions (which can be asynchronous), and feeds the results back into the LLM's next prompt. This cycle—Model Request -> Tool Execution -> Result Feedback -> Next Model Response—is the fundamental agent loop [20:30].

To demonstrate real-world capability, a date tool is added. After initial failures due to declaration errors, the agent successfully identifies and returns the current date when prompted [21:45]. This proves the agent has memory and can now interact with its environment via tools.

The discussion then shifts to advanced standardization using Model Context Protocol (MCP). MCP is a standardized protocol designed for agents to interact with services across different models, moving beyond manually writing every tool. The speaker introduces an MCP client that connects to external MCP servers, such as one providing file system access or a calculator [29:00]. This client translates the structured declarations from the MCP server into the specific JSON schema format required by the LLM.

The final evolution involves Agent Skills. Unlike MCP servers which provide explicit tools, skills use "progressive disclosure." A skill starts with a simple header (name and description), and the agent only loads what it needs to know initially [35:00]. If the agent requires more detail, it requests the rest of the skill content. This allows for complex capabilities without immediately overloading the model's context window. The speaker shows how an agent can be instructed via a system instruction to use its available skills, leading it to execute commands like using a bash tool provided through a skill [40:30].

Key Takeaways

  • An AI agent is fundamentally defined as an application that achieves goals by observing its environment and acting upon it using tools [2:50].
  • The "agent harness" is the orchestration code built around an LLM, responsible for managing instructions, memory, and reasoning loops [4:30].
  • Basic text generation can be upgraded to a chat agent by passing all previous conversation messages back into the model with each new prompt to establish short-term memory [10:00].
  • Tool use requires defining functions using JSON schema, which tells the LLM when and how to invoke external code based on its description and parameters [15:00].
  • The core of an agent is a loop that checks for function calls from the model; if present, it executes those functions externally and feeds the results back into the model for synthesis [20:30].
  • Model Context Protocol (MCP) standardizes how agents interact with external services by providing standardized tool declarations across different models [29:00].
  • Agent Skills utilize "progressive disclosure," allowing an agent to discover complex capabilities incrementally rather than loading all available tools at once, thus saving context window space [35:00].
  • The future of AI development is shifting from just having powerful models to building robust "agent harnesses" that guide and protect these models while enabling goal achievement [45:00].

Practical Notes

  • Tools Definition: Tools must be declared with a name, a description (to help the agent decide when to use it), and a parameters object structured as JSON schema. For example, an addition tool requires two number properties, 'a' and 'b' [15:00].
  • Agent Loop Implementation: The core logic involves a `while` loop that checks if the model response contains function calls. If so, it maps over these calls, executes the corresponding external functions (which can be asynchronous), and then sends the results back to the LLM in the next turn [18:30].
  • MCP Integration: To use MCP tools, an agent client must connect to an MCP server transport (e.g., standard IO or streamable HTTP). The client translates the server's tool declarations into the specific JSON schema format required by the target LLM [29:00].
  • Agent Skills Usage: Skills are activated via a single tool that reads and understands skill markdown files from a designated directory. A system instruction must be provided to inform the agent about the available skills so it knows when to invoke the activation tool [40:30].
  • Risks/Caveats: Over-enthusiastic use of MCP servers with hundreds of tools can quickly exhaust context windows and increase input token costs due to the size of the tool declarations [35:30].
