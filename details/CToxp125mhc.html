What This Video Is About

This video demonstrates how to leverage the new ChatGPT desktop application to automate personal workload prioritization and recurring task management. The core concept revolves around setting up automated workflows that aggregate information from various sources, allowing users to stay informed about urgent deadlines and necessary actions without manual checking. By integrating services like Slack, email, and calendar, a dedicated "Chief of Staff" chat can synthesize daily context for the user.

Furthermore, the video illustrates how these local automation setups can be scaled into persistent, cloud-based Workspace Agents. This allows repeatable processes, such as feedback triage from multiple sources, to run continuously for an entire team via ChatGPT or Slack, effectively turning one-off scheduled tasks into always-on digital assistants.

Detailed Walkthrough

The presenter begins by introducing the capability of using the new ChatGPT desktop app to manage and prioritize ongoing workloads [0:02]. The initial setup involves creating a dedicated chat named "Chief of Staff" within the application [0:10]. This specific chat is configured to connect to external services, namely Slack, email, and calendar, enabling it to pull together all urgent tasks, upcoming deadlines, scheduled meetings, and any other relevant context required before starting the workday [0:13].

To ensure continuous awareness of priorities, a scheduled task is established within this "Chief of Staff" chat. This task is set to send the user a daily brief every weekday, providing a summary of what requires attention during morning routines like having coffee [0:28].

The demonstration then shifts focus to managing incoming feedback, which in the presenter's context relates to improving technical onboarding at OpenAI. Feedback originates from several platforms, including Slack, Notion, and Drive [0:38]. To handle this volume of input, a second scheduled task is created within a separate chat designated as "Feedback Triage" [0:52]. This weekly task runs in the background, its function being to generate a Google Doc that summarizes all collected feedback before posting this summary directly into the team's Slack channel [1:02].

The workflow currently operates locally on the presenter's laptop. However, the video explains how this local triage flow can be converted into a scalable Workspace Agent [1:08]. The process for conversion is described as simply asking ChatGPT to perform the transformation [1:22]. Once converted and moved to the cloud via this agentization process, the team gains the ability to use this automated workflow directly from either the ChatGPT interface or within Slack, enabling 24/7 operation for the entire team [1:32].
