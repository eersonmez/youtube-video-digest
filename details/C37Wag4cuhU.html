The concept of a polymorphic mindset in leveraging agentic AI for software development workflows.

NDC Conferences · 2026-06-26T14:11:23+00:00 · Polymorphic Mindset - Marina Santos Haugen - NDC AI 2026

What This Video Is About

This video discusses the concept of adopting a "polymorphic mindset" when leveraging agentic Artificial Intelligence (AI) within software development workflows. The central argument is that the true value derived from agentic AI does not reside in the agents themselves, but rather in the design and quality of the overall workflow surrounding them. The speaker posits that many agentic projects fail due to process issues rather than technological limitations.

The discussion emphasizes a necessary shift for organizations: transformation requires redesigning processes and redefining roles, rather than simply deploying new AI tools onto existing structures. Furthermore, it stresses that while AI offers significant power—giving developers "superpowers"—the human developer remains fully responsible for all code output generated by the agents. The core message is that the synergy between human skill and AI capability creates genuine value.

Detailed Walkthrough

The presentation begins with an introduction to the speaker's polymorphic career path, moving from business development to experiment management and finally into software engineering, illustrating her ability to adapt across different contexts [0:00]. She defines a "polymorphic mindset" as the ability for one entity (like herself) to take on many forms or roles when context demands it, drawing parallels to polymorphism in programming where one interface can have different behaviors [0:15].

Traditionally, product development involves distinct roles: Product Manager (sets direction/prioritizes backlog), Developer (build/test/refactor), and Designer (UI/UX/user journey mapping) [2:30]. The speaker argues that agentic AI makes teamwork less sequential and more collective, shifting value from task execution to system shaping. Citing reports, she notes that over 40% of agentic projects fail due to process issues, not technology [5:00].

A crucial point made is the need for organizational redesign; deploying tools without workflow changes won't lead to transformation [7:30]. The speaker warns against viewing AI as a replacement or just a tool that generates code easily. She stresses that one bad line of planning or research can result in thousands of lines of bad code, making the quality of the *plan* far more important than syntax alone [12:00-14:30].

To address this, she proposes treating every task as a test of the workflow's efficacy, introducing the concept of "autonomy of a workflow." This process follows four phases: exploration, planning, coding, and verification. The system is powered by several blocks:

  • Configuration (personal adjustments for optimal flow).
  • Skills (recipes for tasks written in markdown files).
  • Plugins (bundles that include skills and hooks).
  • MCP servers (Model Context Protocol, acting as a USB port allowing agents to interact with external tools).

Configuration Details: To manage prompts, she advises housekeeping on permissions using slash commands or enabling auto mode [12:50]. She uses "inside boxes" to see the agent's reasoning and enables explanatory output styles via `/config` [13:40]. Context management is also key; she uses techniques like "compact and clear" to maintain a good context window [14:30].

MCP Servers: These connect agents to external services. Examples include plugins for Figma (design/ideation), Linear/Jira/GitHub issues tracking, Notion, security analysis via an MCP server for security analysis, and Playwright for browser testing [15:00-17:00]. She highlights Context 7 as a tool providing fresh documentation beyond LLM training cutoffs. SRP (Security Analysis Plugin) works with Cloud Code to scan output against OWASP Top 10 vulnerabilities [18:30].

Skills and Plugins: The speaker advocates for building skills rather than just agents, viewing them like specialized applications [21:00]. She demonstrates custom skills, such as "pixel theme" for consistent aesthetic generation. Pre-built skills include `/batch` (for migrating from Vue to React), `/simplify`, and `/loop`.

Advanced Workflows:

  • Superpowers: A toolkit involving 14 different skills covering planning (e.g., brainstorming that outputs an HTML page for visualization), implementation (using test-driven development/sub-agent driven development with work trees), and quality (verification for completion, which runs tests and type checks) [20:30].
  • Feature Depth: A seven-phase pipeline starting with discovery, moving to exploration where 2-3 agents triangulate data, followed by a mandatory clarification phase. It suggests up to three different architectures before implementation begins [24:00].

The workflow culminates in the verification phase, which is critical because plausible code does not equal working code. She combines several elements here: using a writer and reviewer (not the same session), Verification Before Completion (running tests/lint), and employing external judges like Gemini Flash via OpenRouter for UI feedback [50:30].

Finally, she details an autonomous flow skill that automates the entire process—self-verification, code review by six agents, browser testing against acceptance criteria, followed by a "commit dance" skill to cluster commits into a pedagogical narrative before final PR submission. She concludes with using Conductor to run multiple cloud agents in parallel for rapid upskilling and context gathering [54:00].

Key Takeaways

  • AI is fundamentally a tool; the user's mindset and skill are what drive meaningful results, not the technology itself.
  • The workflow design must be treated as the primary product being tested for quality when using agentic AI.
  • Organizations achieve transformation by redesigning their processes, not just by deploying new tools.
  • A structured process involving exploration, planning, coding, and verification is essential for effective AI utilization.
  • The human developer retains full responsibility and ownership over all code output generated by AI agents.
  • Value shifts from executing discrete tasks to shaping complex systems through well-designed workflows.
  • Polymorphism—the ability to adapt roles (e.g., PM, Designer, Developer) within the workflow—is a key skill for leveraging AI effectively.
  • Verification is non-negotiable; without it, AI can produce plausible but functionally incorrect code.

Practical Notes

  • Process: Follow an iterative process: Exploration -> Planning -> Coding -> Verification. This should be viewed as testing your workflow's autonomy.
  • Configuration: Use slash commands (e.g., `/permissions`, `/config`) for housekeeping and to enable explanatory output styles.
  • Tools/Components: Utilize MCP servers to connect agents to external tools like Figma, Linear, GitHub Issues, Notion, Playwright, and Context 7 for up-to-date documentation.
  • Skills vs. Agents: Prioritize building custom skills (recipes) over relying solely on general agent capabilities, as skills allow for domain expertise embedding.
  • Security: Use MCP servers like SRP to scan code against OWASP Top 10 vulnerabilities during the development cycle.
  • Workflow Orchestration: Consider using tools like Conductor to run multiple cloud agents in parallel for tasks such as gathering context or generating PRs across different aspects of a feature.
