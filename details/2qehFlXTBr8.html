The benefits of non-intrusive programming techniques in software development.

NDC Conferences · 2026-07-23T11:30:24+00:00 · On the benefits of non-intrusiveness - Patrice Roy - NDC Toronto 2026

What This Video Is About

This video discusses the architectural trade-offs in software development when extending existing code or adding new functionality to types where modification is not possible or undesirable. The central theme is advocating for "non-intrusive" programming techniques—methods that allow developers to add behavior or express constraints without altering the original source code of a type.

The speaker contrasts these non-intrusive approaches with traditional, intrusive methods like adding base classes or implementing interfaces directly onto existing types. While intrusive solutions are often taught early on and provide clear structure, they can impose significant costs, especially when applied retroactively to large, established codebases that require extensive refactoring and recompilation. Non-intrusiveness offers freedom and independence in code evolution, which is crucial when dealing with external libraries or immutable types.

Detailed Walkthrough

The discussion begins by illustrating a common problem: needing to query properties of fundamental types (like the maximum value of an integer) that are not natively supported across different languages or type systems [0:00]. The speaker notes that while some languages provide this via built-in classes, others require specific mechanisms. In C#, for instance, support was added later via a contract like IMaxValue<T>, but applying such interfaces retroactively to older classes requires modification of the original code [5:00].

The core argument is that when a language lacks a native mechanism to express a desired capability, resorting to an intrusive solution forces costly refactoring. The speaker contrasts this with traditional object-oriented patterns like using base classes or interfaces, which are teachable and provide clear structure for new developers [10:00]. However, these methods require modifying the original code.

The video then explores advanced language features that enable non-intrusive extensions. In C#, extension methods allow adding behavior to types externally without altering their definition [5:00]. The speaker demonstrates attempts to enforce constraints using custom interfaces (e.g., IConvertibleTo<T, U>). Initially, these attempts sometimes resulted in complex or unintuitive error messages because the language's idioms did not perfectly align with the desired constraint expression [15:00].

The discussion moves into C++ concepts like unconstrained templates and traits for compile-time reasoning about types. Traits allow developers to infer information (like whether a type is constant, or if it has a `max` function) based on its structure without modifying the type itself [35:00]. This allows for adding "code documentation" that is fully compiled.

The speaker highlights modern language features like C++ concepts and C#' s extension methods as solutions to these long-standing problems, providing syntactic constraints that allow developers to express intent—such as requiring a type to be "addable"—without altering the underlying types [50:00]. Finally, the video touches upon using wrappers or decorators to achieve similar goals when direct code modification is impossible, balancing design trade-offs between intrusive and non-intrusive approaches [1:00:00].

Key Takeaways

  • Non-intrusive solutions grant freedom and independence from existing code structures by adding functionality externally.
  • Intrusive changes to existing types can necessitate costly refactoring and recompilation, especially in large projects.
  • Generic programming concepts allow for expressing requirements without modifying original types.
  • Extension methods (e.g., in C#) enable the addition of behavior to types from outside their definition.
  • Traits (in C++) allow compile-time reasoning about a type's properties (like size or conversion capabilities) based on its structure, acting as external documentation.
  • Modern language features like concepts provide syntactic constraints that check if an expression is valid for a given type without requiring the type to implement specific interfaces.
  • Wrapping existing classes in wrapper types can allow new functionality to be added externally when direct modification is prohibited.
  • The choice between intrusive and non-intrusive methods involves significant design trade-offs regarding code purity versus implementation complexity.

Practical Notes

  • C# Extension Methods: These are used to add functions that appear as member functions of a type but are actually defined outside the class, provided they only use public members [5:00].
  • C++ Traits: Used for compile-time introspection; they allow querying properties of types (e.g., `std::numeric_limits`) without altering the type definition itself [35:00].
  • C++ Concepts: These are language features that allow constraints to be placed on template parameters, effectively stating "this function works if T is addable," which checks validity at compile time [50:00].
  • Wrapper/Decorator Pattern: When direct modification is impossible, wrapping the original class in a new type that implements necessary interfaces allows external code to interact with it as if it had those capabilities [1:00:00].
