Agentic Engineering: Notes from Building and Shipping with Agents

NDC Conferences · 2026-07-01T13:52:59+00:00 · Agentic Engineering: Notes from Building and Shipping with Agents - Mats Oustad - NDC AI 2026

What This Video Is About

This video provides an in-depth overview of agentic engineering, detailing the evolution of using coding agents over the past 18 months. The central theme is a shift away from simple prompt engineering toward building robust, durable workflows that blend product management principles with development practices. While initial excitement around "vibe" coding has waned, the focus must now be on strategic direction and systematic quality assurance to ensure agent-generated output is safe and valuable in production environments.

The discussion emphasizes that while iteration speed has dramatically increased—reducing proof-of-concept time from years to minutes—the human role evolves into that of a conductor or orchestrator. This requires developers to focus heavily on defining goals, structuring processes using reusable "skills," and implementing rigorous verification steps, such as automated testing and cross-model checks, rather than just writing code themselves.

Detailed Walkthrough

The presentation outlines several key shifts in the field of agentic engineering. Initially, the focus was on prompt engineering, where users spent significant time crafting perfect prompts using tools like ChatGPT or Claude in a browser [5:20]. This has shifted; now, developers shape plans collaboratively with coding agents.

A major advancement noted is the introduction of plan modes in tools like Cloud Code [6:03], which has evolved into more powerful workflows via plugins such as Superpowers. These plugins facilitate brainstorming and maintain a human-in-the-loop process between steps, allowing users to build context together rather than just feeding raw context into the tool [6:40]. This makes iteration much cheaper and faster.

A critical shift involves prioritizing deterministic tools over pure inference when possible. The speaker argues that expensive token usage should be avoided for tasks that can be solved deterministically using scripts or CLI tools [8:21]. For example, extracting emails from a data file using an LLM took about 20 seconds and used many tokens; in contrast, using the JQ tool was substantially faster and consumed far fewer tokens [11:36].

The concept of "skills" is presented as essential for making repetitive tasks reproducible. A naive approach to fixing a failing test might work once, but when complex policies (like banker's rounding) are involved, a skilled agent using specific skills—such as "pricing policy research"—will produce a more robust and correct fix [19:37]. The speaker demonstrates a custom "research skill" that pulls data from sources like YouTube via the YT DLP CLI to generate a verdict file containing key facts and applicability for knowledge base integration [21:06].

Regarding output presentation, the speaker advocates for HTML over Markdown when presenting agentic loops or workflows because it is visually superior and allows for interaction, which can be further enhanced by creating skills that dictate presentation style [24:13]. Furthermore, using tools like Anthropic's playground plugin enables contextual feedback on generated content, allowing users to iterate on slides or web outputs directly within the session [26:17].

The workflow is moving toward orchestrating multiple agents across projects. The speaker uses `tmux` to manage separate sessions for different projects, increasing bandwidth by running many agents concurrently while maintaining fresh context for each session [30:00]. Finally, quality assurance requires automated verification; the agent must verify its own work through tests and validations rather than relying solely on human review of what it *said* versus what it *did* [30:47].

Key Takeaways

  • Agentic engineering has evolved from simple prompt engineering to shaping complex plans with coding agents.
  • Prioritize using deterministic CLI tools (like JQ) over pure LLM inference for tasks that can be solved programmatically to save tokens and time.
  • "Skills" are crucial for making repetitive, common problems reproducible within agent workflows.
  • The role of the developer is shifting from code writer to conductor or orchestrator, focusing on goal setting and process design.
  • Verification must be automated through tests and validations; agents need mechanisms to verify their own output quality.
  • Utilizing multiple models for cross-checking plans can help uncover subtle errors before human review.
  • For complex tasks, workflows are becoming capable of running autonomously for extended periods (e.g., 21 minutes or over an hour).
  • Security requires careful human oversight in the looping process to prevent data leakage, as model output might not always reflect actual actions taken.

Practical Notes

  • Tool Usage: Favor CLI tools like JQ for structured data manipulation over LLM inference when dealing with JSON/data files [11:36].
  • Workflow Enhancement: Implement skills (e.g., "pricing policy research") to enforce specific business logic or processes consistently across agent runs [19:47].
  • Local vs. Cloud Models: Consider running smaller, local models (like Gemma 4) on developer machines for simple tasks to reduce reliance on expensive cloud APIs, provided the hardware supports it (e.g., 32-64GB RAM) [49:04, 55:31].
  • Process Management: Use tools like `tmux` to manage multiple agent sessions concurrently across different projects while maintaining isolated contexts [30:00].
  • Security/Quality Check: When planning complex tasks, split large plans into discrete issues in an issue tracker (like GitHub) and use a second model to review the plan before execution [32:15].
