What This Video Is About

This video introduces the new capabilities of ChatGPT, demonstrating its evolution from a simple question-answering interface into a comprehensive work assistant capable of executing complex, multi-step tasks across various professional domains. The core theme is the ability to delegate real work to ChatGPT by connecting it with a user's existing tools, local files, and even directly interacting with their operating system or browser.

The demonstration showcases how this new functionality integrates into personal productivity (acting as a Chief of Staff), marketing (generating brand visuals and ad campaigns), data science (analyzing metrics and flagging issues), and software engineering (diagnosing bugs using 'appshots'). This shift allows users to hand off substantial work, receive updates, and return to the task when necessary, effectively making ChatGPT an integrated part of professional workflows.

Detailed Walkthrough

The presentation begins by introducing a new version of ChatGPT available on web, desktop, and mobile [0:03]. The primary change is that it moves beyond conversation to actively performing work by connecting to user tools like Calendar, Slack, and Drive [0:11]. Initially, the presenter demonstrates preparing for their day using these connected services [0:25], which gathers notes and meeting preparation information.

The concept of automation is then introduced through a "Chief of Staff" persona [0:52]. The user sets up an automated workflow to receive daily briefings on what they need to prepare, along with real-time updates like sports scores for specific teams (e.g., the German team during the World Cup) [1:00]. This automation persists without needing constant prompting.

The functionality extends across devices; work started on the web can continue seamlessly on mobile [1:15]. A significant shift occurs with the desktop app, which allows ChatGPT to interact with local files and projects [1:22]. The presenter explains that this enables context-heavy tasks where the user hands off the work and checks back later.

Three detailed use cases are presented:

  • Marketing: For a fictional neobank named Hudson, ChatGPT was used with brand guidelines and a logo to generate various visuals for a mobile app launch site [1:54]. The process then evolved into creating mood boards for an ad campaign and subsequently generating a presentation deck based on those concepts [2:13], resulting in a ready-to-use deck for a launch review.
  • Data Science: After the launch, the user tasked ChatGPT with synthesizing data from various sources—metrics, feedback, conversations, and plans—into a cohesive analysis [2:40]. The system automatically applied relevant skills to generate insights on activation rates and onboarding funnels. It specifically flagged an issue in Mexico, suggesting a potential phone onboarding problem based on the data [3:11]. This raw data was then transformed into an interactive dashboard visualized within the in-app browser [3:24], which could be annotated with one-line summaries. The final step involved using "Sites" to publish this dashboard directly from ChatGPT, ensuring daily metric updates and sharing it via a link in a team channel [3:45].
  • Engineering (Codex): To address the flagged issue, the user switched to Codex. First, they used the Simulator to bring up the app and captured an "appshot" by pressing Cmd-Cmd [4:17]. An appshot is described as a smarter screenshot that provides ChatGPT with the necessary context of the running application. The system then performed background Computer Use, acting directly within the user's applications (not just via cursor control) to compare phone onboarding in Mexico versus Germany [4:36]. This confirmed users were stuck entering valid Mexican phone numbers. Finally, the user instructed ChatGPT to prepare a fix for this validation bug; the resulting code difference could be reviewed directly within Codex before being published as a Pull Request (PR) [5:09].

Key Takeaways

  • ChatGPT has evolved into an agent capable of performing multi-step work by integrating with external tools.
  • The system supports persistent automation, such as setting up daily briefings via a "Chief of Staff" persona that runs independently after initial setup.
  • Integration spans personal productivity (Calendar, Slack, Drive), marketing (visual generation, mood boards), data analysis, and software engineering.
  • The desktop application allows ChatGPT to interact with local files and projects, enabling context-aware task delegation.
  • "Appshots" are a key feature that functions as an intelligent screenshot, providing the AI with the necessary visual and contextual information about a running application for diagnosis.
  • When dedicated plugins are unavailable, ChatGPT can utilize background Computer Use to interact directly with the user's applications or browser extensions.
  • Outputs like data dashboards can be published directly into web pages using the "Sites" feature for easy sharing and automated updates.
  • The workflow in engineering involves diagnosing issues via appshots, having the AI prepare a fix (diff), reviewing it in Codex, and publishing a PR.

Practical Notes

  • Tool Integration: ChatGPT can connect to personal tools such as Calendar, Slack, and Drive [0:25].
  • Automation Setup: Workflows can be set up to run automatically without repeated prompting (e.g., daily briefings) [1:07].
  • Local Context: The desktop app is necessary for interacting with local files and projects [1:31].
  • Appshots: This feature captures the state of an application, providing context for AI actions; it is triggered by specific commands (e.g., Cmd-Cmd) while using a Simulator [4:17].
  • Background Computer Use: When no dedicated plugin exists, ChatGPT can operate directly on the user's computer or browser via extensions to copy data, browse sites, or check workflows [4:36].
  • Output Publishing: The "Sites" feature allows users to publish generated content, such as dashboards, directly from ChatGPT and set up automated updates for those published links [3:45].
  • Engineering Workflow: The process involves identifying an issue (e.g., validation bug), having the AI prepare a fix in Codex, reviewing the diff, and publishing a PR [5:16-5:20].
