The video showcases Dometrain, an AI-first, hands-on learning platform for software developers.

Nick Chapsas · 2026-07-21T11:45:11+00:00 · I Built the Future of Learning for Developers

What This Video Is About

Nick Chapsas presents Dometrain as a developer-learning platform rebuilt around active practice rather than passive video watching. He says the platform spent its first several years primarily offering video courses, but its learning experience changed substantially over the preceding six months. The new approach puts learners directly into browser-based coding exercises, system-design diagrams, Git workflows, and AI-assisted lessons.

The central argument is that developers learn better by doing the work themselves, while still having access to the guidance of an experienced instructor. Dometrain’s AI agents are intended to provide that guidance inside the learning environment: they can offer hints, execute a confirmed solution, explain changes, run tests, quiz the learner, and relate lessons to the learner’s existing progress. Chapsas describes this as learning “with the agent,” rather than simply asking an agent for answers.

The video also introduces video-course assistance, personalized learning paths, public profiles, achievements, skill assessments, and an MCP integration for Pro members. It explains the platform’s broad product direction and subscription boundaries, but it does not provide the promised technical deep dive into the underlying architecture, agent implementation, costs, or security. Chapsas says those subjects will be covered in a separate video.

Detailed Walkthrough

From a Video Library to an Interactive Learning Environment

At the beginning, Chapsas explains that he has spent roughly seven years teaching .NET and C-related development and has been running Dometrain for about three and a half years. He says the platform previously consisted mainly of video courses in which an author taught a particular topic, such as a programming challenge or a language feature. He reports that approximately 73,000 engineers are actively using the platform.

His reason for changing the product is both pedagogical and technological. He considers hands-on work the best way to learn, while describing a good video course as the next-best option because it provides instruction from someone with senior-level knowledge. He also argues that software development is becoming increasingly AI-agent-driven, so learning platforms need to adapt to that environment rather than treating AI as a separate, optional tool [0:00-2:14].

Browser-Based C# Practice with Monaco and Automated Tests

The first major change was the release of a fully hands-on course in January. The example shown is a roughly 20-hour getting-started course for C# that can be completed in the browser. Learners read instructions, work through language features step by step, write code, and run it without opening a separate IDE [2:16-3:09].

The editor is based on Monaco, the editor engine behind VS Code. It provides IDE-like editing and autocomplete, including support for writing code such as `Console.WriteLine`. The learner can run the code directly in the browser, and the platform executes a set of automated tests. Chapsas describes a sample exercise that runs six test cases and prints their results in the console.

The intended workflow is incremental: read the explanation, implement the requested change, run the code and tests, inspect the result, and continue to the next feature or exercise. The platform therefore combines instructional text, an editor, execution, and assessment in the same environment.

System-Design Exercises Using an Interactive Canvas

Dometrain later expanded the hands-on model beyond programming exercises. A system-design course lets learners create an architecture diagram on a canvas [3:17-3:57].

Chapsas gives the example of placing a client and a service on the canvas, connecting them, and labeling the connection with a protocol such as HTTP. The service might represent an API service, while the client represents the component making the request. The learner is expected to construct the system rather than simply view a completed diagram.

The platform checks whether the diagram contains required elements and can explain why those elements are needed. This turns system design into an interactive exercise involving nodes, relationships, protocols, and architectural reasoning. The goal is not only to write code but also to understand system architecture and the decisions represented in a diagram.

The introductory C# course is described as approximately 20 hours, while the introductory system-design course is approximately four hours. Chapsas says both are available free of charge: a user can create an account, claim them, and keep access permanently [3:59-4:18].

Subscription Tiers and the Browser-Based Git Terminal

Chapsas describes the Light subscription as including the hands-on courses, interview questions, and system-design courses. He says it averages around £10 per month. He contrasts it with Pro, which he says is more expensive partly because Dometrain wants to compensate authors fairly according to video watch time.

The hands-on environment also includes an in-browser Git terminal, intended to give learners a broader development workflow without requiring a locally configured machine [5:04-6:01]. The terminal supports Git configuration, status checks, branches, source-control information, and command-line operations. The examples mentioned in the video include:

  • Initializing a repository.
  • Setting the Git username and other configuration.
  • Running `git status`.
  • Adding files with `git add .`.
  • Committing through either the interface or `git commit -m "check in"`.
  • Pushing changes after configuring a remote URL where required.
  • Reviewing history with `git log`.

Chapsas says the Git material amounts to roughly seven hours and that the relevant features can be practiced directly in the platform. This makes Git part of the same browser-based learning loop as coding and system design.

The AI Agent as a Guided Learning Partner

The platform is described as AI-agent-first. Chapsas says the agents have access to course context, integrations, the learner’s profile, and retrieval-augmented generation, or RAG, behind the scenes [6:04-6:24].

In a coding exercise, a learner who is stuck can ask for a hint rather than immediately requesting the answer. The example hint suggests using a built-in decimal addition method. This supports a back-and-forth conversation in which the agent nudges the learner toward the solution.

A separate solve action requires two confirmations. Chapsas says this is intended to discourage learners from using the agent as a way to cheat through every exercise. Once confirmed, the agent can write the code, run the tests, show the resulting feedback, and explain the generated code line by line. It also presents the changes as diffs, allowing the learner to see what was added and deleted [6:49-7:27].

The important distinction for Chapsas is that the agent should operate inside a learning environment. His formulation is that the learner should learn “with the agent,” not simply learn from an agent that provides unexplained answers. The surrounding instructions, tests, diffs, and explanations are meant to make the agent’s intervention inspectable.

Agent-Assisted System-Design Solutions

The same interaction model applies to diagram exercises. A learner can ask the agent to solve a system-design task, after which the platform animates the construction of the diagram. The agent adds the diagram elements step by step and explains why each element was added [7:27-8:30].

The transcript describes a displayed explanation for one part of the diagram: the user entry point is restricted to communicating with the service tier so that it does not have direct access to sensitive data or credentials. This is presented as an example of the contextual explanation attached to a solution step. The learner can hover over the relevant item and inspect the rationale.

The agent also operates in the Git lessons. Chapsas describes it initializing a repository, setting the Git username configuration, progressing through subsequent commands, and providing a summary after completing the lesson. The learner can then ask the agent to quiz them on what happened or what the lesson was intended to teach [8:31-9:10].

AI Assistance for Video Courses

Video-course access is described as a Pro feature rather than a Light feature, but Chapsas says the agent is also available while watching videos [9:17-10:30]. Its video-related functions include:

  • Answering questions about commands or concepts.
  • Recapping the lesson or the portion already watched.
  • Generating a quiz about the material.
  • Searching for a topic within the current course or other courses and returning a timestamp and link.
  • Extracting code shown on screen.
  • Explaining what is visible in the current video frame.

For code extraction, the agent captures the current frame and produces the code shown on the screen so the learner can copy it into another environment. For screen explanations, the agent examines the frame at the current playback position and adds explanatory annotations. Chapsas gives an example involving .NET 10 results, methods with a `new` constraint, performance overhead, and associated numbers.

These functions are intended to make a video course interactive. Instead of pausing the video to search elsewhere, the learner can ask for a recap, retrieve a timestamp, extract a code sample, or request an explanation of the material currently being discussed.

Personalized Recommendations and Learning Paths

Dometrain also has a global learning agent that uses the learner’s profile and completed work to make recommendations [10:33-11:42]. When asked what to learn next, it can inspect the learner’s existing progress and suggest a relevant course.

Chapsas demonstrates a more specific request for a learning path covering ASP.NET Core APIs, Minimal APIs, Dapper, Entity Framework Core, and unit testing. The agent checks the available catalog and the learner’s progress, then creates a sequence of milestones. The resulting path includes areas such as EF Core, Dapper, testable code, and unit testing.

The learner can view and start the generated path, ask the agent to rework it, or share it with a friend or team so that others can follow the same sequence. The path therefore serves both as a personalized curriculum and as a shareable plan.

Profiles, Achievements, and Progress Sharing

The platform includes daily activities, achievements, public profiles, and a leaderboard. A public profile can show activity, completed lessons, course completions, the date the user joined Dometrain, and collected achievements [11:43-12:24].

Chapsas says these details are configurable. Users can adjust what is shown or turn the public profile off entirely. The platform also provides a real-time dashboard showing who is completing lessons, which may support group or team-based learning.

The social features are presented as an extension of the learning workflow: progress is not limited to private course completion but can also be represented through achievements, activity, and shared learning paths.

Dometrain MCP for Pro Members

Chapsas briefly describes an MCP integration available to Pro members [12:28-12:47]. A user can connect Dometrain’s MCP to an external coding agent and ask it to use Dometrain’s knowledge while building a project.

His example is a request to build clean architecture using Dometrain’s material. The external agent calls the Dometrain MCP and builds the project according to the way Dometrain’s authors teach the subject. The video does not explain the MCP’s implementation, data flow, pricing impact, or security model; Chapsas reserves those details for the separate architecture discussion.

Skill Assessments and Automatically Generated Study Plans

A newer feature is the skill score assessment [12:53-14:08]. Chapsas says only around three skills are supported at the time of recording, with plans to add more. Before beginning, the learner describes their level as new, comfortable, experienced, or expert. That selection determines an initial ELO-style rating.

The assessment presents code or questions, including prompts asking what a piece of code will do. The demonstration uses 20 questions. Chapsas intentionally answers randomly, receives a novice rating of 891, and gets only two answers correct. The result is an illustration of the assessment process rather than a meaningful evaluation of his actual ability.

Afterward, the learner can share the score on LinkedIn or connect it to their real profile. The transcript says there is a 10-minute opportunity to retake the test to improve the result. The platform identifies knowledge gaps and lets the learner review every question, including why an answer was correct or incorrect, relevant misconceptions, and advice for improvement. It then generates a study plan recommending courses intended to address those gaps.

What This Video Does Not Cover

Chapsas says the feature list shown is incomplete and that more functionality is in development. Although he initially mentions architecture diagrams, operating costs, and implementation choices, the video focuses on the product experience rather than those technical details.

He announces a separate video covering how the agents were implemented, the overall architecture, security, and related operational considerations [14:34-14:44]. Those subjects should not be inferred from the demonstrations in this video.

Key Takeaways

  • Dometrain is shifting from a primarily video-based course library to an environment centered on hands-on browser practice.
  • The coding environment uses Monaco, the editor engine behind VS Code, with autocomplete, code execution, console output, and automated test cases.
  • The platform extends hands-on learning to system design through an interactive canvas where learners create, connect, label, and validate architecture diagrams.
  • An in-browser Git terminal supports repository initialization, configuration, status checks, staging, commits, pushes, logs, and related practice.
  • The AI agent can provide hints before revealing a solution, while the solve action requires confirmation to reduce casual cheating.
  • Agent-generated code is accompanied by diffs, test execution, feedback, and line-by-line explanations.
  • Diagram solutions are also explained step by step, including the rationale for adding each component or relationship.
  • Video-course agents can recap lessons, create quizzes, search for topics with timestamps, extract code from a frame, and explain material visible at the current playback position.
  • A global learning agent uses the learner’s profile and catalog progress to generate personalized learning paths, such as a path involving ASP.NET Core APIs, Minimal APIs, Dapper, EF Core, and unit testing.
  • Public profiles, achievements, leaderboards, daily activity, and shareable learning paths add social and team-oriented progress features, with controls for disabling public visibility.
  • Pro members can connect Dometrain’s MCP to an external coding agent so projects can be built using Dometrain’s instructional approach.
  • Skill assessments use an initial self-selected level, question results, identified gaps, explanations, and a generated study plan to direct future learning.

Practical Notes

A transcript-supported way to use the platform is to begin with the free introductory courses, read each exercise’s instructions, implement the requested change in the Monaco editor, and run the included tests. If blocked, the demonstrated progression is to request a hint first, inspect the suggestion, and only use the confirmed solve action when necessary. After a solution is generated, the useful learning artifacts are the diff, test results, line-by-line explanation, and summary rather than the generated code alone.

For system-design work, the learner creates the architecture directly on the canvas. A representative exercise can involve a client, an API or service node, an HTTP-labeled connection, and any required supporting components. The platform checks the diagram and explains why required elements belong in the design.

The Git workflow shown includes initializing a repository, setting the username and other configuration, checking status with `git status`, staging with `git add .`, committing with `git commit -m "check in"`, reviewing history with `git log`, and pushing when a remote URL has been configured. Some lessons may therefore require setting up a remote before the push step can work.

The free introductory C# course is described as approximately 20 hours, and the introductory system-design course as approximately four hours. Light is described as including hands-on courses, interview questions, and system-design courses for approximately £10 per month. Pro is described as the tier that adds video-course access and Dometrain MCP access. The transcript’s promotional pricing is time-sensitive and internally inconsistent: it mentions a `light50` code for 50% off a first month or year, then later mentions a three-day offer of approximately $5.99 for a month. Current terms would need to be verified directly rather than assumed from the video.

The main learning decision is whether the learner needs only hands-on and interview-oriented material or also wants video courses and MCP access. The transcript also makes clear that the solve function can complete an exercise, so learners who want to preserve the challenge should use hints, explanations, tests, and quizzes before requesting a full solution.

The platform uses the learner’s progress and profile to personalize recommendations. Users who do not want their progress exposed can adjust the public-profile settings or disable the profile. The video also shows an agent using course knowledge and profile context, but it does not state the retention, privacy, or data-handling policies behind those features.

The technical architecture remains an open question in this video. The only architecture-related implementation details explicitly mentioned are the use of Monaco, a diagram canvas, a browser Git terminal, AI agents, RAG, integrations, and learner-profile context. Details about services, model providers, isolation, agent permissions, operating costs, and security are deferred to the planned follow-up video.
