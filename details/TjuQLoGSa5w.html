What This Video Is About

The central topic of this video is a shift in focus from traditional, routine programming tutorials to AI-assisted engineering, product building, and advanced technical review within software development. The speaker explains that his reduced output of standard coding videos is not due to demotivation caused by Artificial Intelligence (AI), but rather because he has been deeply immersed in understanding how AI is fundamentally changing the industry. He posits that while AI reduces the bottleneck of writing code, it significantly elevates the importance of skilled review, robust testing, architectural design, and critical judgment from the developer.

The video serves to explain this new direction by showcasing work being done around his platform, Dometrain. The core argument is that in an era where LLMs possess vast amounts of knowledge, the scarce skill is no longer typing code, but rather knowing precisely what to ask the AI for, rigorously reviewing its output, and effectively testing it. He emphasizes that fundamental coding skills remain crucial because they enable developers to detect flaws in AI-generated code and guide future outputs more effectively.

Detailed Walkthrough

The speaker begins by addressing his recent absence from regular programming videos [0:01] and clarifies that this shift is tied to diving deep into the impact of AI on software development over the past six to eight months [1:37]. He dismisses the idea that AI is universally detrimental, suggesting it will likely persist in most industries, though perhaps evolving toward local models as frontier models become too expensive for all companies [1:05], [1:12].

His new focus is specifically on helping C# developers navigate this landscape [2:11], moving away from simply teaching "how to write better C code with AI," which he deems an insufficient angle given the current reality. Instead, his work centers on how to attack complex projects using these tools [2:30].

He highlights several areas of development and teaching:

  • Workshops: He is running VIP coding for production workshops globally, leveraging everything learned while building Dometrain with AI [3:04], [3:15]. These workshops teach how companies want to use AI in production code.
  • The Role of Skilled People: A key insight shared is that the bottleneck has shifted from writing code to having enough skilled personnel capable of reviewing the code instructed by AI [4:16], [4:20]. He stresses that AI should function as a projection of the developer; it must produce code one would be capable of reviewing [4:25].
  • Planning and Review: The speaker released "my real work skill," which details his planning process when working with LLMs to ensure there is no ambiguity, aiming to prevent the LLM from producing "slop" [5:08], [5:37]. He stresses that proper planning is where most time will be spent.
  • System Design: Recognizing that LLMs cannot fully convey complex architectural concepts through chat alone, he released a free category of system design courses [6:21]. This tool allows users to read instructions, define requirements, and receive real-time feedback on designs involving services, databases, message queues, API gateways, etc. [7:04], [7:33].
  • Interview Practice: A recently released feature is a mock interview tool designed for low-stakes practice [9:06]. Users can select a role (e.g., junior C# developer) or paste a job description to generate an interview that assesses technical knowledge, company culture fit, and more [9:43], [10:03]. The system allows customization of the interviewer's persona (friendly coach, tough, masculine/feminine voices) and can integrate code challenges directly into the conversation [10:21], [10:54].

Key Takeaways

  • AI is changing software development by shifting the primary skill requirement from manual coding to expert review, testing, and architectural judgment.
  • The speaker's focus has moved toward helping C# developers effectively leverage AI in complex project completion rather than basic syntax tutorials.
  • AI should be treated as an extension of the developer; its output must be something a human could reasonably write and review.
  • Effective collaboration with LLMs requires rigorous planning to eliminate ambiguity before generation begins.
  • System design is a critical area where dedicated tooling (like Dometrain's courses) is necessary because LLM chat alone cannot convey complex architectural visualizations.
  • The mock interview tool allows users to practice for any role by generating tailored technical and cultural assessments based on job descriptions.
  • Hands-on, practical learning—where developers code things themselves before instructing AI—is highly valued for building the necessary foundation to review AI output critically.

Practical Notes

  • Dometrain: The platform is being used to build and teach these new concepts. Users can access free beginner system design courses [6:35] and utilize the mock interview feature (which requires Dometrain Pro membership for full use) [14:17].
