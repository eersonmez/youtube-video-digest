Smart Home Solver rebuilds a smart home from the network upward with Home Assistant, Z-Wave, scenes, and resilient automations.

Smart Home Solver · 2026-06-13T16:44:50+00:00 · How I’m Starting a New Smart Home from Scratch!

What This Video Is About

This video details the process of rebuilding a smart home system from first principles, emphasizing that a robust underlying network is the essential foundation upon which all other smart features must rest. The creator advocates for a highly customized setup using Home Assistant as the central automation platform to overcome the limitations of proprietary ecosystems like Apple, Amazon, or Google.

The core philosophy presented is to prioritize solving high-annoyance problems in frequently used areas first, rather than simply adding devices randomly. By carefully selecting hardware and structuring automations around reusable "scenes," the goal is to achieve a smart home experience that removes daily irritations while maintaining the reliability of traditional, non-automated systems.

Detailed Walkthrough

The process begins with establishing a reliable network infrastructure [0:19]. The creator advises taking an ISP's all-in-one modem/router and putting it into bridge mode to prevent double NAT issues, warning that using the ISP device as the router will lead to frequent crashes [0:34–0:57]. For routing hardware, while UniFi is mentioned, a budget TP-Link mesh system can suffice for beginners. The setup involves upgrading to components like a 10 gig switch and U7 Pro XG wireless access points [1:18–1:29]. A key architectural decision is minimizing Wi-Fi usage; devices like computers are hardwired through Ethernet to low-voltage switches, which feed into the main 10 gig switch where smart home hubs reside [2:05–2:27].

The choice of automation platform is Home Assistant [2:55], praised for its ability to integrate nearly every smart device and its open-source nature. The selection criteria for initial devices are based on two factors: the severity of the annoyance, and whether the area receives high traffic [4:17–4:23]. In the living room, the first target was solving shadow issues caused by ceiling fans and lights when both were active [4:48–4:50]. The solution implemented involved using a Zooz ZEN32 smart switch [5:08], which allows for customization of button functions (e.g., bright white light, orange dim, blue fan control) and is reliable because it uses Z-Wave technology [5:16–5:41].

For pendant lights, the creator opted for high-quality "dumb" bulbs paired with Lutron dimmer switches instead of smart bulbs to ensure 100% reliability when using the physical switch [6:48–7:02]. The preference for communication protocols is strictly ordered based on perceived reliability: Z-Wave first, then Zigbee, then Thread, with Wi-Fi being the last choice [7:32–7:51].

Before writing complex automations, the workflow dictates creating reusable scenes—such as "bright," "dim," or "TV watching"—to serve as a single source of truth for all controls [7:59–8:46]. For instance, five living room scenes were established. Automations are then built upon these scenes; one example addresses the fan/light conflict by creating an automation with two triggers (fan on, light on) and using conditions to turn off the conflicting device [8:54–10:02].

A critical implementation lesson involves Home Assistant's automation mode. Initial testing failed because rapid button presses were ignored; this was resolved by changing the automation mode from single to parallel mode, allowing multiple actions to run simultaneously when triggered rapidly [10:32–11:07]. Furthermore, schedules can be made intelligent; for example, an evening light schedule is conditioned to skip changes if the Apple TV is playing content, preventing glare on the screen [11:37–11:54]. Finally, desk-light automation was achieved by triggering based on the computer turning on and connecting to a second screen, avoiding unreliable motion sensors [12:18–12:40].

Key Takeaways

  • Network First: A reliable network foundation is paramount; this requires bridging ISP modem/routers and prioritizing wired connections for high-value devices.
  • Platform Choice: Home Assistant is recommended over proprietary systems due to its extensive compatibility (Matter, Z-Wave, etc.) and open-source nature.
  • Prioritization Strategy: Start automation efforts by targeting the most annoying problems in the areas of highest traffic usage.
  • Scene Management: Always build reusable scenes first; this centralizes settings so that updating a scene updates all associated buttons and automations simultaneously.
  • Protocol Hierarchy: When selecting devices, prioritize Z-Wave over Zigbee, then Thread, with Wi-Fi being the least preferred due to reliability concerns.
  • Automation Mode: In Home Assistant, ensure complex interactions involving rapid inputs are set to "parallel mode" rather than the default "single mode."
  • Predictability Over Sensors: Automating based on existing device states (e.g., computer power state) is more predictable and less prone to false triggers than relying solely on motion sensors.

Practical Notes

  • Network Setup: Change ISP all-in-one device to bridge mode [0:36]. Consider using UniFi or a budget TP-Link mesh system for routing.
  • Device Selection Workflow: Check for Z-Wave versions first, then Zigbee, then Thread, and finally Wi-Fi [7:32–7:40].
  • Switch Implementation: Zooz ZEN32 is cited as an example of a customizable, wired Z-Wave switch that allows multiple functions per button.
  • Dimmer Reliability: For critical lighting control where physical switches must work 100% of the time, use smart dimmer switches (e.g., Lutron) paired with high-quality dumb bulbs [6:48–7:02].
  • Automation Logic: Complex interactions can be managed in Home Assistant using multiple triggers and conditions within a single automation entity to control device states simultaneously.
