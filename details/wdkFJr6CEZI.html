An overview of underutilized Web APIs that are advancing web experiences toward native application parity.

NDC Conferences · 2026-06-24T14:13:21+00:00 · Web APIs You Should Be Using in 2026: Unlocking Next-Level Web Experiences - Nhlanhla Lucky Nkosi

What This Video Is About

This video provides an overview of several modern and often underutilized Web APIs designed to significantly advance the capabilities of web applications, pushing them toward parity with native mobile and desktop applications. The central theme is bridging the experiential gap between browser-based software and platform-specific applications, which is particularly relevant in markets where mobile usage dominates, such as Africa [8:35].

The speaker argues that while traditional critiques often cite security or access limitations in browsers compared to native apps, modern APIs are closing this divide by granting web experiences direct access to system resources like the file system and device hardware. The discussion covers tools for improved user experience (UX), enhanced reliability even under poor network conditions, and massive performance gains for computationally intensive tasks like on-device AI inference.

The presentation advocates for developers to leverage these APIs not as a replacement for native development entirely, but as powerful building blocks that allow web platforms to deliver richer, more robust experiences without forcing users into cumbersome download/upload cycles or relying solely on URLs for sharing content.

Detailed Walkthrough

The speaker begins by establishing his perspective: he dislikes mobile apps and is passionate about making the web platform indistinguishable from a native experience [4:25]. He introduces several key APIs that facilitate this convergence.

File System Access API: This API addresses the cumbersome workflow of traditional file handling in browsers, which typically involves giving a file input, uploading it to memory/server, manipulating it, and then prompting a download [10:00-10:24]. The File System Access API allows web applications to read, write, and save files directly onto the user's machine without requiring constant uploads or downloads [10:36-10:49]. Key benefits include an enhanced UX where users can work directly on a file, enabling offline capabilities for Progressive Web Apps (PWAs), and performance enhancements by reducing network calls [11:07-11:43]. The API utilizes native user prompts like "show open file picker" and is secure by design, requiring explicit permission from the user to access directories or files [12:06-12:29]. A demo shows that changes made in the browser are saved directly into the local file system upon saving, without an upload/download cycle [13:40-14:15].

Web Share API (Level Two): This addresses the limitation of sharing web content by only providing a URL. For applications with authentication layers or sensitive data, simply copying and pasting a link is insufficient for sharing the actual artifact [16:26-16:38]. The Web Share API Level Two allows developers to package the actual content—such as an image, video, or text file (a blob)—and pass it directly into the operating system's native share tray [18:32-19:04]. This brings the native sharing experience into the browser and simplifies development by eliminating the need for custom share buttons. The key differentiator from Level One is that Level Two shares the data object itself, not just a URL [19:11-19:15]. A demonstration shows successfully sharing a local test file to a colleague via Teams using this API [20:47-22:17].

Contact Picker API: This API offers a privacy-preserving method for web applications to access user contact details, which is highly valuable in developing markets where phone numbers are critical for services like wallet payments or group buying (e.g., Pinduoduo/Timu) [23:06-24:12]. Unlike traditional methods requiring users to manually copy and paste contacts, the Contact Picker allows users to explicitly choose which parts of their contact information—such as name, phone number, or company—to share with the application via an OS-level picker [25:00-25:33]. This adds user trust by giving them granular control over data access.

Background Fetch API: This API solves reliability issues associated with large downloads or uploads when a user closes their browser tab or experiences network interruptions [27:26-28:14]. By executing fetch commands in a background thread managed by the service worker, the download or upload continues even if the main tab is closed. Once complete, the API updates a message that the application can hook into to process the data [30:10-30:37]. This is particularly useful for PWAs operating in low connectivity areas [29:18-29:31].

WebGPU API: For computationally intensive tasks like AI inference, WebGPU offers massive performance improvements over previous technologies. It provides a direct layer to the underlying GPU from the browser without requiring developers to manage complex kernel systems [33:38-33:53]. The speaker demonstrates that switching computation from WebGL to WebGPU can result in astronomical performance gains, such as moving from 500 frames per second to over 8,000 frames per second in a 3D render without code changes [37:08-37:19]. This capability allows for running light AI models directly in the browser, which is crucial given the rising costs of general-purpose cloud inference [38:00-38:52].

Future/Emerging APIs: The speaker also mentions several upcoming or proposed APIs that could further close the gap between web and native experiences, including Web NFC API for secure, low-interaction payments and sharing, Web Biometrics for in-browser biometric authentication, and Health Connect API for integrating fitness tracking data [39:42-41:09].

Key Takeaways

  • Modern Web APIs are actively closing the feature gap between web applications and native mobile/desktop apps.
  • The File System Access API allows direct reading, writing, and saving of files on a user's machine, eliminating the need for constant upload/download cycles.
  • Web Share API Level Two enables sharing actual file content (blobs) through native share trays rather than just providing a URL.
  • Contact Picker API provides a privacy-preserving way to access necessary contact details from a user's device contacts with explicit user consent.
  • Background Fetch API ensures that large downloads or uploads continue reliably even if the browser tab is closed, improving reliability for PWAs in low connectivity areas.
  • WebGPU offers astronomical performance boosts by allowing direct GPU access from the browser, enabling on-device AI inference without constant server reliance.
  • The convergence of these APIs allows web developers to build richer experiences that mimic native functionality while maintaining cross-platform deployment benefits.

Practical Notes

  • File System Access API: Requires user permission via explicit prompts; only functions over HTTPS; supported in Chrome, Edge, and Opera (Safari/Firefox are working on support).
  • Web Share API Level Two: Requires a user gesture to initiate sharing; functionality varies based on the host device's ecosystem (e.g., AirDrop on macOS vs. Windows integration).
  • Contact Picker API: Currently has limited support, specifically mentioning Chromium-based browsers and Android web views.
  • Background Fetch API: Relies on Service Workers to execute fetch commands in a background thread; useful for offline uploads when connectivity is restored.
  • WebGPU API: Requires careful resource management as it can be computationally intensive; offers massive performance gains over WebGL for graphics and computation.
  • Development Strategy: The speaker suggests being "craftier" about system design, determining which tasks require heavy general-purpose cloud AI versus those that can be handled by smaller, optimized models running directly in the browser via GPU access.
