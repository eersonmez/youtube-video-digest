Ten tips to enhance productivity when using AI-assisted coding tools.

NDC Conferences · 2026-07-01T14:36:06+00:00 · 10 tips to level up your ai-assisted coding - Aleksander Stensby - NDC AI 2026

What This Video Is About

This video provides a comprehensive guide on how developers and non-developers can move beyond passively accepting outputs from AI coding agents, such as Claude Code. The central theme is shifting the interaction with AI from simple prompting to active collaboration, adopting a "pair programming mindset." As generative AI capabilities advance rapidly—with significant leaps in models like Opus and new benchmarks—the focus must shift from the ability to write code to defining clear human intent, goals, and desired outcomes.

The presentation details several advanced techniques for managing these powerful tools effectively. Key concepts include optimizing the finite context window, leveraging portable "skills" to enforce preferences or grant new capabilities, structuring work using planning modes, and understanding how bottlenecks are shifting from coding ability to human oversight and verification. The goal is to become a more effective collaborator with AI rather than just a consumer of its output.

Detailed Walkthrough

The speaker begins by establishing that the most crucial mindset when using AI coding tools is adopting a pair programming approach [0:00]. This means actively collaborating with the AI instead of lazily throwing prompts and accepting results, which risks making users less capable over time. A concept introduced is "compound engineering," where the goal is to have the AI learn from every mistake made during collaboration so it never repeats that error again [5:00].

A critical technical aspect discussed is context management, as the context window of any AI model is a finite resource [10:00]. The speaker likens this window to a glass; system prompts, agent configuration files (like agent.md or Claude.md), tool definitions, and conversation history all fill this space. While modern windows are large (e.g., around 200,000 tokens, with some reaching one million), results deteriorate when approaching 70-80% capacity [10:00]. Users can monitor this using commands like {slash} status line in Claude Code or by checking usage summaries in tools like Cursor [10:00].

To manage rules and preferences across different AI harnesses, the speaker suggests a "hack": placing rules in both agent.md and Claude.md files so that either tool picks them up [15:00]. Memory is another concept, allowing state persistence between sessions via files like memory.md, which must be actively managed and pruned to prevent critical rules from being lost in excessive information [15:00].

The speaker strongly advocates for using "skills" as a portable solution [20:00]. Skills are essentially text or markdown files that can teach the AI preferences (e.g., workflow, taste) or grant it new capabilities not inherent to the base model (e.g., enabling image generation via an API skill). A key recommendation is to build custom skills rather than downloading existing ones [25:00].

A major shift in focus involves moving from writing code to defining clear goals and outcomes [30:00]. Plan mode, available in many solutions like Cursor, allows for safe exploration of a codebase by having the AI create plans. It is recommended to persist these plans externally so the AI can verify its own work against the initial plan later [25:00].

As coding becomes more reliable, the bottleneck shifts from writing code to human intent and verification—the "human bottleneck" [30:00]. To combat this, users should challenge existing team processes (like code reviews or sprint planning) when integrating AI. Furthermore, staying engaged requires using formats beyond simple text; presenting plans in interactive HTML files can create a more engaging feedback loop with the AI [35:00].

The process of verification is evolving toward "agentation," where one agent verifies another's work [40:00]. The ultimate goal is to move towards defining an outcome (using features like {slash} goal) and trusting the system, which operates as a multi-agent loop where agents check each other's results until the goal is met.

Finally, for scaling productivity, users should explore "loop engineering," which involves having agents store state or tasks across sessions to perform continuous work autonomously [1:00:00]. For advanced use cases, Ultra Code allows Claude to write its own harness on top of existing tools [1:00:00].

Key Takeaways

  • Adopt a pair programming mindset when using AI coding agents; actively collaborate rather than passively accepting outputs.
  • Treat the context window as a scarce resource and optimize for high-impact tokens, monitoring usage via commands like {slash} status line.
  • Utilize "skills" (portable text/markdown files) to teach the AI your preferences or equip it with new capabilities.
  • Shift focus from writing code lines to defining clear goals and desired outcomes for the AI agent.
  • Manage memory and rules actively; do not let configuration files like Claude.md grow disproportionately, as this can hold back model improvements.
  • Invest time in planning and describing intent using features like Plan Mode or creating persistent markdown plans that the AI can verify against.
  • Challenge existing team processes (e.g., code reviews) to adapt to a world where AI handles much of the coding.
  • Use subagents to engineer effectively infinite context windows, though be mindful not to over-specify them if newer models are capable of defining agents on the fly.
  • Employ hooks for deterministic guardrails, ensuring actions happen outside of inference based on specific events.
  • Explore loop engineering by having agents store state or tasks across sessions to enable continuous work.

Practical Notes

  • Context Management: Use {slash} status line (in Claude Code) or check usage summaries in tools like Cursor to monitor context window consumption. If auto-compaction occurs, guide it using {slash} compact with specific instructions.
  • Rule Enforcement: Rules can be placed in agent.md or Claude.md; placing them in both allows cross-tool compatibility.
  • Skill Creation: The best skills are built by the user *with* the AI through iterative feedback. Use the skill creator to build and test these, potentially leveraging its built-in evaluation suite.
  • Goal Setting: Instead of describing *how* to do something, define the desired outcome using features like {slash} goal to allow the agent to determine the steps.
  • Tool Integration (MCP/CLI): Equip your AI with necessary tools (e.g., CLI commands) so it can perform actions on your behalf; this is crucial for verification and task execution.
  • Verification: Give the AI the ability to verify its own work by providing it with testing tools or instructing it to check output against a persisted plan.
  • Optimization: For token usage, run {slash} /usage in the terminal to get a summary of what is consuming tokens, including specific skills.
