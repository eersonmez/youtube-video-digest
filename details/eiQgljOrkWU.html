OpenAI has announced significant updates to Codex, integrating it into ChatGPT with new features for developers.

OpenAI · 2026-07-14T23:30:19+00:00 · The new Codex for developers

What This Video Is About

This video details a significant integration of Codex into the ChatGPT application, presenting developers with a unified and enhanced development environment. The core message is that Codex has evolved from a standalone tool into an integrated platform capable of handling complex, long-running projects by leveraging new frontier models and advanced automation features. It aims to streamline the entire software development lifecycle, from initial idea conception to deployment and maintenance, all within one interface.

The updates focus heavily on increasing the model's reasoning capabilities through features like Ultra mode in GPT-5.6 Sol, enabling it to manage complex tasks autonomously. Furthermore, the integration introduces powerful tools for interacting with application contexts—such as using "appshots" from simulators—and dramatically simplifies deployment via a new "Sites" feature that handles infrastructure automatically.

Detailed Walkthrough

The presentation begins by announcing that Codex and ChatGPT are merging into one unified application [0:19]. For developers, this means the familiar Codex experience is now housed within ChatGPT alongside a dedicated work agent [0:31]. The speaker notes over 150 updates have been shipped in the last two months.

The first major update discussed is the model itself: GPT-5.6 Sol, which is available to everyone and designed to tackle difficult problems over extended periods [0:47]. A key enhancement is the introduction of "Ultra mode" when used with the Goal command, which grants Sol a significantly larger reasoning budget for challenging projects [0:59]. However, this increased capability comes with the caveat that Ultra mode consumes token limits faster [1:07].

A major workflow improvement is Codex's ability to automatically distribute work across subagents without explicit prompting [1:11]. The user can monitor this collective effort or follow individual agents working on a single goal in parallel. Sol also demonstrates advanced computer and browser interaction capabilities, which the presenter highlights as one of the most exciting features [1:26]. For instance, when developing an iPhone app, Codex can take an "appshot" of the Simulator by pressing Cmd-Cmd on Mac, effectively capturing both the screen content and the application context. This allows Codex to interact with the simulator itself—for example, clicking around in the background while the user continues working elsewhere [1:40, 2:04].

The platform supports visual iteration through an in-app browser where users can interact with code examples, such as a simple paper plane game built with GPT-5.6 [2:10]. If modifications are needed, the user can point and click to annotate specific parts of the interface, providing Codex with visual context for requested changes [2:22]. The browser functionality has also been expanded to support applications requiring logins, including passkey support [2:32]. Furthermore, developers can now switch into a diff view to make precise inline edits directly in the code if they prefer working visually over having Codex generate the change [2:42-2:51].

For deployment, the "Sites" feature is introduced as a solution for taking built applications live. Now available to everyone (previously limited to Sites for Enterprise), this allows users to publish any web application created with Codex directly to Sites [3:18]. This service bundles hosting, authentication, persistent database support, and file storage into one package, allowing developers to move from concept to a full-stack application without managing infrastructure [3:29-3:40].

Organization is further improved through task management. Codex can now manage its own tasks; for example, it can find five bugs listed in Linear, create isolated worktrees for each, and pin the most critical ones [4:11]. These coordinated tasks can reference one another. Moreover, context from other conversations within ChatGPT can be brought directly into Codex by using an @mention, eliminating the need for manual copy-pasting [4:39-4:48].

Mobile capabilities have seen significant upgrades. On mobile devices, users can now create, search, open, and manage tasks directly from a conversation [5:02]. Reviewing changes is improved with filtering options (staged, unstaged, branch, compare branches) available on the go [5:13]. Additionally, users can connect directly to an SSH host from their phone to start new tasks remotely [5:20].

Finally, day-to-day workflow within the application has been smoothed out regarding pull requests (PRs). Codex operates in context with PRs; if a task is working on a branch with an open PR, the summary pane surfaces it automatically [5:50]. Users can address issues or ask Codex to fix failures directly within this context. A dedicated Pull Requests tab allows users to explore code changes, leave inline comments, complete reviews, and mark their own PRs as ready for merging without leaving the application environment [6:10-6:18].

Key Takeaways

  • Codex is now unified within the ChatGPT application, providing a single interface for development tasks.
  • GPT-5.6 Sol is the new frontier model, capable of handling long and complex problems.
  • Ultra mode in GPT-5.6 Sol provides an expanded reasoning budget for the most challenging projects but increases token usage rate.
  • Codex can autonomously split a single goal into parallel tasks managed by subagents.
  • "Appshots" allow Codex to capture context from simulators, enabling it to interact with running applications visually.
  • The "Sites" feature enables one-click deployment of web applications, including built-in hosting and database support.
  • Codex can manage its own tasks, allowing users to organize multiple independent pieces of work (e.g., finding bugs in Linear).
  • Context from other ChatGPT conversations can be brought into Codex workflows via @mentions, eliminating copy-pasting.
  • Mobile upgrades allow task management and direct SSH connections from a phone.
  • Codex integrates PR workflow directly into the app context, allowing for reviews, fixes, and merges without leaving the interface.

Practical Notes

  • Model Use: To utilize the enhanced reasoning capabilities, use GPT-5.6 Sol combined with the Goal command and enable Ultra mode [0:59]. Be mindful of token consumption when using Ultra mode [1:07].
  • Context Capture: For simulator interaction, use the Cmd-Cmd shortcut on Mac to take an "appshot" [1:40]. This provides Codex with both screen data and application context.
  • Deployment Workflow: To deploy a web application built with Codex, issue the command "publish this to Sites" [3:29]. This automatically provisions hosting, authentication, database, and file storage.
  • Task Coordination: When managing multiple issues (e.g., from Linear), instruct Codex to create separate tasks and isolated worktrees for each identified bug [4:11].
  • Mobile Development: Mobile users can initiate new tasks by connecting directly to an SSH host via the mobile application [5:20].
  • PR Workflow Integration: When working on a branch with an open Pull Request, Codex will automatically surface it in the summary pane; this allows for inline commenting and merging without navigating to GitHub.com [5:50-6:18].
